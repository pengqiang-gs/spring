# Spring概述

标签： Spring

## Spring介绍
Spring是最受欢迎的企业级Java应用程序开发框架，可以用来创建性能好，易于测试，可重用的代码。Spring框架的核心特性是可以开发任何Java程序，但是在JavaEE平台上构建Web程序则需要扩展。Spring启动通过POJO编程模型来促进良好的编程实践。

## Spring框架优势
- Spring可以让开发人员使用POJOs开发企业级应用程序
- Spring在一个单元模式中是有组织的
- Spring可复用的组件很多
- Spring更易于测试
- Spring对于JavaEE中难用的API进行了封装
- Spring提供了轻量级的IoC容器
- Spring提供了一致的事务管理接口

## 依赖注入
Spring的依赖注入（DI）是控制反转（IoC）的主要模式。当编写一个复杂的JavaEE程序时，应用程序应该尽可能地独立于其他Java类来增加Java类的可重用性。当进行单元测试时，可以使它们独立于其他类来进行测试。依赖注入有利于将这些类粘合在一起，并且在同一时间让它们保持独立。

> 依赖注入理解
比如类A依赖于类B，那么在使用类A时Spring将类B通过IoC注入到类A中。
依赖注入可以通过向构造函数传递参数使用，也可以通过使用`setter`方法`post-construction`。
后面章节会根据实例详细介绍这部分。

## 面向方面的程序设计
Spring框架的一个关键组件是面向方面的程序设计（AOP）框架。一个程序中跨越多个点的功能被称为**横切关注点**，这些横切关注点在概念上独立于应用程序的业务逻辑，比如日志记录，安全性等例子。

在OOP的编程模型中，关键的单元是类（class），而在AOP的编程模型中，关键的单元是方面。AOP可以帮助程序开发者将横切关注点从它们所影响的对象中分离出来，然而依赖注入帮助程序开发者将应用程序对象从彼此中分离出来。
